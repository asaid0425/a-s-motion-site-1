case study / 03 — linkedin · motion system

ux motion / system.

Turning scattered one-off animations into a coherent motion language — by auditing every flow, breaking each screen into states, and designing only the motion that measurably improves the experience.

role · ux motion + systems scope · product-wide output · motion framework
/ 01

the audit.

Motion already existed across LinkedIn — but as scattered one-offs, every team inventing its own by feel. Before designing anything new, I inventoried every motion in the product and sorted each one by the job it was actually doing for the user.

motion inventory — every existing animation captured, then grouped by intent
/ 02

the method.

I don't treat motion as decoration laid on top. I break every screen into its states — the initial and final state of each interaction — and treat the space between them as the real design surface.

every initial state → final state — the gap between them is where motion lives
01
Map each screen as a set of states — every initial and final state for every interaction in the app.
02
Between every pair of states sits a gap. That gap is an opportunity for motion — I look at all of them, not just the obvious ones.
03
Each candidate answers one question: does it make the product easier to understand or use? Every decision is rooted in UX — never in spectacle.
04
What survives is motion that does a job. Everything else is cut.
/ 03

four types of motion.

Every motion that earns its place does one of four jobs. Naming them gave the team a shared language — and a test for whether a given animation belongs at all.

type 01
gesture confirmation
Motion that answers the user's input the instant it happens — a press depresses, a swipe tracks the finger, a toggle snaps home.
ux valueTrust — the user never has to wonder whether their action registered.
type 02
motion affordance
Motion that reveals what an element can do before it is touched — a gentle nudge, a peek, a settle that says “I can move.”
ux valueDiscoverability — the interface teaches its own gestures without a tooltip.
type 03
hiding latency
Motion that covers the wait — skeletons, progressive reveals, and transitions that fill the time the system needs to respond.
ux valuePerceived speed — the wait feels shorter than it actually measures.
type 04
spatial design
Motion that keeps the world coherent — shared elements, depth, and direction so screens feel connected, not swapped.
ux valueOrientation — the user always knows where they are and where things went.
/ 04
